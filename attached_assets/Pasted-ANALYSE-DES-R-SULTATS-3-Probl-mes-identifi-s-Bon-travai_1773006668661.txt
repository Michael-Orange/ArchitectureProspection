PROBLÈME 2 - SerpAPI retourne beaucoup d'articles web (pas des cabinets)

Exemples dans les résultats :
- "La construction – Elementerre" → Article SUR Elementerre (pas le site officiel)
- "Les Spécialistes des Briques..." → Article générique
- "D'architecture en terre crue" → Page blog

CORRECTION NÉCESSAIRE :

Ajoute un filtre dans `scrapeGoogleSearchSerpAPITool` pour **ignorer les pages d'articles** :

```typescript
// Après avoir récupéré les résultats SerpAPI

const excludePatterns = [
  '/la-construction/',  // Pages d'articles
  '/focus/',
  '/bonne-pratique/',
  '/case/',
  'climate-chance.org',  // Agrégateurs à filtrer
  'hardel-lebihan.com',  // Cabinets français sans présence locale Sénégal
];

for (const result of data.organic_results || []) {
  const url = result.link.toLowerCase();
  
  // Ignorer les pages d'articles
  if (excludePatterns.some(pattern => url.includes(pattern))) {
    console.log(`Skipped article: ${result.title}`);
    continue;
  }
  
  // Prioriser les sites avec domaine propre (pas des sous-pages)
  const isHomepage = url.match(/^https?:\/\/[^\/]+\/?$/);
  const isMainSite = !url.includes('/focus/') && !url.includes('/case/');
  
  if (isHomepage || isMainSite) {
    firms.push({
      name: result.title,
      website: result.link,
      ...
    });
  }
}
```

Garde uniquement les **domaines principaux** des cabinets (ex: worofila.com, dnasarl.com).
